VenueHub

Multi-Tenant SaaS Venue Management Platform

Product Requirements Document

Version: 1.0 | Date: April 13, 2026 | Status: In Development

1. Executive Summary

VenueHub is a comprehensive, multi-tenant SaaS platform designed to help banquet halls, event venues, and wedding venues streamline their operations. It provides complete end-to-end management of venues, events/bookings, staff, and revenue tracking with advanced analytics, calendar views, and reporting capabilities.

Key Value Propositions:

  • Centralized venue and booking management
  • Multi-tenant architecture with complete data isolation
  • Real-time revenue analytics and reporting
  • Intuitive calendar-based event scheduling
  • Role-based access control for team collaboration
  • Built for scale with modern cloud-ready architecture

2. Vision & Goals

Vision

To become the leading all-in-one management platform for event venues, enabling venue owners and managers to focus on delivering exceptional experiences while automating administrative tasks.

Goals

  1. Goal 1: Enable venues to manage bookings efficiently with 80% reduction in manual work
  2. Goal 2: Provide actionable analytics for revenue optimization and business decisions
  3. Goal 3: Support team collaboration with role-based permissions and multi-user access

4. Core Features

4.1 Authentication & Authorization

4.1.1 User Registration

  • Email-based registration with unique tenant creation
  • Password hashing with bcrypt (salt rounds: 12)
  • JWT token generation with 7-day expiration
  • First registered user automatically assigned "admin" role

4.1.2 User Login

  • Email and password authentication
  • JWT token generation with user role included
  • Persistent token storage in localStorage
  • Automatic session refresh on page load

4.1.3 Session Management

  • Automatic role extraction from JWT
  • Bearer token validation on all protected routes
  • Logout functionality clearing stored tokens
  • Protected routes require valid authentication

4.2 Venue Management

  • Create: Add new venues with capacity, pricing, and details
  • List: View all venues with filtering and search
  • Update: Edit venue information and settings
  • Delete: Remove venues (soft/hard delete options)

4.3 Event/Booking Management

  • Create: Book events with client details, dates, and amounts
  • List: View all bookings with status and date information
  • Update: Modify event details and status
  • Delete: Cancel or remove events

4.4 Staff Management

  • Add: Onboard staff members with roles and contact info
  • List: View team members with departments and status
  • Update: Modify staff information and assignments
  • Delete: Remove staff members

4.5 Tenant Management

  • Create: Organizations are auto-created on registration
  • Get Info: Retrieve organization details
  • Update: Modify organization name, logo, and settings

5. Advanced Features

5.1 Role-Based Access Control (RBAC)

5.1.1 Role Definitions

Role Description Permissions
Admin Full system access Manage users, venues, events, staff, settings, analytics
Manager Day-to-day operations View/create/edit venues, events, staff
Staff Limited access View assigned events and basic info
Guest Minimal access Read-only access to specific events (future)

5.2 Analytics Dashboard

5.2.1 Summary Metrics

Key Metrics Displayed:
  • Total Revenue: Sum of all event totalAmount
  • Advance Collected: Sum of all advanceAmount received
  • Balance Pending: Total Revenue - Advance Collected
  • Total Guest Count: Sum of all guestCount across events
  • Event Status Breakdown: Counts for pending, confirmed, completed, cancelled

5.2.2 Revenue by Venue

Groups revenue and event count by venue to show which venues generate the most revenue and help with resource allocation decisions.

5.2.3 Event Trends

Time-series data grouped by month showing seasonal patterns, growth trends, and revenue forecasting insights.

5.2.4 Status Breakdown

Visual breakdown of event statuses to track booking pipeline health and identify bottlenecks.

5.3 Calendar View

5.3.1 Monthly Calendar

  • Interactive calendar grid showing all events for the month
  • Previous/Next month navigation
  • Current date highlighted (yellow)
  • Selected date highlighted (blue)

5.3.2 Event Display

  • Events shown on respective dates
  • Event names displayed with truncation
  • "+" indicator when multiple events on same date
  • Status badges with color coding

5.3.3 Quick Actions

  • Click date to see details and create events
  • Sidebar form for quick event creation
  • Event list for selected date
  • Status badges for quick identification

Database MySQL 9.6+
ORM Drizzle 0.29+
Validation Zod 3.22+
Authentication JWT + bcrypt Latest

6.2 Database Schema

Core Tables:
  • tenants: Organizations with isolation boundaries
  • users: User accounts with roles and authentication
  • venues: Banquet halls and event spaces
  • events: Bookings and reservations
  • staff: Team members and designations

6.3 Multi-Tenant Isolation

  • Every query filtered by tenantId
  • Foreign key constraints link all data to tenants
  • No cross-tenant data leakage at database level
  • JWT includes tenantId to prevent spoofing
  • All mutations validate tenantId matches context

6.4 Security Implementation

Current Security Measures:
  • ✅ JWT with 7-day expiration
  • ✅ Password hashing with bcrypt (12 salt rounds)
  • ✅ Bearer token in Authorization header
  • ✅ Role-based access control
  • ✅ Multi-tenant isolation
  • ✅ Input validation with Zod schemas
  • ✅ SQL injection prevention (ORM-based)
  • ✅ CORS with whitelist (localhost:5173, 5174)

10. Deployment & Scaling

Development Environment

  • Frontend: Vite dev server (localhost:5174)
  • Backend: Node.js with tsx (localhost:3001)
  • Database: Local MySQL (localhost:3306)

Production Environment

  • Frontend: Vite build → static hosting (Vercel/Netlify)
  • Backend: Express → Docker container → Kubernetes/Railway.app
  • Database: Managed MySQL (AWS RDS / PlanetScale)
  • CDN: CloudFlare for static assets

Scaling Strategy

  • Horizontal: Load balancer → multiple Express instances
  • Vertical: Database optimization, connection pooling
  • Caching: Redis for session storage (future)
  • Queue: Bull/RabbitMQ for async jobs (future)

11. Getting Started

Local Environment Setup

# Clone repository
git clone https://github.com/yourusername/venuehub.git
cd venuehub

# Install dependencies
npm install

# Setup environment variables
cp .env.example .env.local

# Database setup
npm run db:push

# Start servers
npm run dev
